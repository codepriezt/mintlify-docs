        #### ðŸ“Œ Steps on how to create a new user and password and disable root user login

        * Log in to your VPS as root and create a new user via the terminal

        ```bash
        ssh root@your_server_ip
        ```

        * Add a new user: Replace new\_username with the username you want

        ```bash
        adduser newusername
        ```

        * Set a password for the new user:

        ```bash
        codepasswd newusername
        ```

        * Add the new user to the sudo group to assign administrative privileges so they can execute commands with elevated  privileges:

        ```bash
        bashCopy codeusermod -aG sudo newusername
        ```

        Now that we have successfully created a user, we process to the next step&#x20;

        ðŸ“ŒDisable Root SSH Login:

        To improve security, itâ€™s a good practice to disable root SSH login.&#x20;

        * Open the SSH configuration file:

        ```bash
        nano /etc/ssh/sshd_config
        ```

        * Find the line that says PermitRootLogin and change it to:

        ```bash
        PermitRootLogin no
        ```

        * Change the SSH Port:

        While you are editing the SSH configuration file (/etc/ssh/sshd\_config), change the SSH port from the default 22 to your preferred port (e.g  Port 2222)

        * Restart the SSH Service:

        After making the changes, save and close the file, then restart the SSH service to apply the changes:

        ```bash
        systemctl restart ssh
        ```

        Allow the New SSH Port Through the Firewall:

        If you are using a firewall, make sure to allow the new SSH port:&#x20;

        * For UFW:

        ```bash
        ufw allow 2222/tcp
        ```

        * Find the line that says PermitRootLogin and change it to:

        ```bash
        PermitRootLogin no
        ```

        <Note>
          Make sure to replace 2222 with the port number you selected.
        </Note>

        Test SSH with the New User and Port:&#x20;

        Note: Before logging out of the root session, open a new terminal and test SSH access with the new user and port:

        ```bash
        ssh -p new_port new_username@your_vps_ip
        ```

        Disable Root Login Completely (Optional):&#x20;

        Once you are able to log in with the new user and new port, you can disable root access altogether: Open the /etc/ssh/sshd\_config file again and ensure these lines are present:

        * Open the /etc/ssh/sshd\_config file again and ensure these lines are present:

        ```bash
        PermitRootLogin no
        ```

        * <u>Restart</u> SSH service again:

        ```bash
        systemctl restart ssh
        ```
